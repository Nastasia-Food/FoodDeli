# Security Policy

## Supported Versions

We release patches for security vulnerabilities for the following versions of FoodDeli:

| Version | Supported          |
| ------- | ------------------ |
| 3.x.x   | :white_check_mark: |
| 2.x.x   | :x:                |
| 1.x.x   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability within FoodDeli, please send an e-mail to our security team at support@rechain.network. All security vulnerabilities will be promptly addressed.

Please do not publicly disclose the vulnerability until we have had a chance to address it.

## Security Measures

We take security seriously and implement the following measures:

- Regular security audits of our codebase
- Secure storage of sensitive data using industry-standard encryption
- Regular updates to dependencies to address known vulnerabilities
- Secure authentication and authorization mechanisms
- Protection against common web application vulnerabilities

## Data Protection

We implement appropriate technical and organizational measures to protect personal data, including:

- Encryption of data in transit and at rest where appropriate
- Regular security assessments
- Access controls and authentication
- Data minimization practices
- Regular staff training on security practices

## Compliance

We comply with applicable data protection laws, including the General Data Protection Regulation (GDPR) for users in the European Union.

## Contact

For any security-related questions or concerns, please contact us at support@rechain.network.
